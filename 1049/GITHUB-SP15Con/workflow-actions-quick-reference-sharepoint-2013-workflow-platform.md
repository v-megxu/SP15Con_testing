# Краткий справочник по действиям рабочего процесса (платформа рабочих процессов в SharePoint 2013)
В этой статье перечислены действия рабочих процессов, которые поддерживаются в текущей сборке SharePoint Designer 2013, а также недоступные действия.
## Действия рабочих процессов в SharePoint Designer 2013
<a name="bkm_WorkflowActions"> </a>

Ниже представлена справка по действиям рабочих процессов, доступным для платформы рабочих процессов SharePoint 2013. Помимо платформы SharePoint 2013, SharePoint Designer 2013 также поддерживает платформу рабочих процессов SharePoint 2010. Действия рабочих процессов для платформы SharePoint 2010 см. в статье  [Краткий справочник по действиям рабочего процесса (платформа рабочих процессов SharePoint 2010)](workflow-actions-quick-reference-sharepoint-2010-workflow-platform.md)
  
    
    

### Основные действия
<a name="bkm_CoreActions"> </a>

Основные действия  это действия, которые выполняются чаще всего и сгруппированы для простоты доступа.
  
    
    

**Таблица 1. Справка по основным действиям**


|**Действие**|**Описание**|
|:-----|:-----|
|Добавить комментарий  <br/> |Позволяет оставлять в конструкторе рабочих процессов информативные примечания для справки. Это особенно полезно, когда с рабочим процессом одновременно работают другие пользователи.  <br/> |
|Добавить время к дате  <br/> |Прибавляет к дате конкретное время в минутах, часах, днях или месяцах (годы не поддерживаются) и сохраняет выходное значение как переменную. Это может быть текущая дата, определенный день или подстановка. Значение "Текущая дата" возвращает полночь в формате UTC.  <br/> |
|Создание словаря  <br/> |Построение переменной Dictionary из пар "ключ-значение".  <br/> > **Примечание**> Словарь использует для хранения данных нотацию JSON.           Дополнительные сведения о переменной Dictionary см. в статье  [Общие сведения о действия словаря в SharePoint Designer 2013](understanding-dictionary-actions-in-sharepoint-designer-2013.md) <br/> |
|Вызов веб-службы HTTP  <br/> |Действует как вызов метода к веб-службе HTTP и возвращает данные в формате JSON. Обычная проверка подлинности поддерживается через RequestHeader.  <br/> Дополнительные сведения о переменной Dictionary см. в статье  [Общие сведения о действия словаря в SharePoint Designer 2013](understanding-dictionary-actions-in-sharepoint-designer-2013.md) <br/> |
|Число элементов в словаре  <br/> |Возвращает число элементов в указанном словаре.  <br/> |
|Выполнить вычисление  <br/> |Выполняет арифметическое вычисление и сохраняет выходное значение как переменную.  <br/> > **Примечание**> В SharePoint 2013 это действие поддерживает только числовой тип **Double**. Целые числа не поддерживаются. Использование оператора "+" (объединение) для строк не поддерживается.           |
|Получить элемент из словаря  <br/> |Возвращает определенный элемент из переменной словаря.  <br/> |
|Записать в журнал  <br/> |Записывает сообщение из списка предопределенных элементов сообщений в список журнала рабочих процессов.  <br/> |
|Сделать паузу в течение интервала времени  <br/> |Останавливает выполнение рабочего процесса на указанный интервал времени, заданный в днях, часах и минутах.  <br/> |
|Сделать паузу до даты  <br/> |Останавливает выполнение рабочего процесса до указанных даты и времени.  <br/> |
|Отправить сообщение электронной почты  <br/> |Автоматически отправляет сообщение электронной почты, которое содержит предопределенное сообщение для пользователя или группы, когда происходит указанное событие рабочего процесса.  <br/> > **Важно!**> Если сайт не добавлен в список "Надежные сайты", то сообщения направляются в папку нежелательной почты Outlook.           |
|Задать время для поля даты и времени  <br/> |Создает временную метку и сохраняет выходное значение в переменной. Вы можете задать время в часах и минутах, а также добавить текущую дату, определенный день или подстановку.  <br/> |
|Изменить состояние рабочего процесса  <br/> |Задает состояние рабочего процесса  <br/> |
|Задать переменную рабочего процесса  <br/> |Задает значение переменной рабочего процесса. Вы также можете использовать это действие, когда рабочий процесс должен назначать данные переменной.  <br/> |
|Перейти к стадии  <br/> |Задает следующую стадию, которой должно быть передано управление потоком.  <br/> |
   

### Действия по координации
<a name="bkm_CoreActions"> </a>

Действия для координации используются для вызова рабочего процесса на платформе рабочих процессов SharePoint 2010. Дополнительные сведения о действия по координации см. в статье  [Общие сведения о действия по координации в SharePoint Designer 2013](understanding-coordination-actions-in-sharepoint-designer-2013.md)
  
    
    

**Таблица 2. Справка по действиям координации**


|**Действие**|**Описание**|
|:-----|:-----|
|Запустить рабочий процесс списка  <br/> |Запускает рабочий процесс списка на платформе рабочих процессов SharePoint 2010.  <br/> > **Примечание**>  С действием "Запустить рабочий процесс списка" связаны следующие проблемы:>  Поле типа "Назначения" невозможно использовать в качестве параметра, если в рабочем процессе SharePoint 2010 есть действие TaskProcess.>  При выполнении нескольких вызовов к одному рабочему процессу SharePoint 2010 результатом будут несколько источников данных в функциях подстановки рабочих процессов SharePoint 2013. Все эти источники данных одинаковы.>  Имена переменных в SharePoint 2013 не могут включать специальные символы, например ? и #. Если рабочий процесс SharePoint 2010 содержит специальные символы, они будут преобразованы в шестнадцатеричный код в рабочем процессе SharePoint 2013.          |
|Запустить рабочий процесс сайта  <br/> |Запускает рабочий процесс сайта на платформе рабочих процессов SharePoint 2010.  <br/> > **Примечание**>  С действием "Запустить рабочий процесс сайта" связаны следующие проблемы:>  Поле типа "Назначения" невозможно использовать в качестве параметра, если в рабочем процессе SharePoint 2010 есть действие TaskProcess.>  При выполнении нескольких вызовов к одному рабочему процессу SharePoint 2010 результатом будут несколько источников данных в функциях подстановки рабочих процессов SharePoint 2013. Все эти источники данных одинаковы.>  Имена переменных в SharePoint 2013 не могут включать специальные символы, например ? и #. Если рабочий процесс SharePoint 2010 содержит специальные символы, они будут преобразованы в шестнадцатеричный код в рабочем процессе SharePoint 2013.          |
   

### Действия со списками
<a name="bkm_ListActions"> </a>

К действиям со списками относятся действия, которые используются для управления списками и их элементами.
  
    
    

**Таблица 3. Справка по действиям со списками**


|**Действие**|**Описание**|
|:-----|:-----|
|Вернуть элемент  <br/> |Возвращает извлеченный элемент. Вы можете возвращать элементы только из библиотеки документов.  <br/> > **Внимание!**> При попытке вернуть элемент, который не был извлечен, происходит сбой рабочего процесса.           |
|Извлечь элемент  <br/> |Извлекает элемент. Рабочий процесс проверяет, возвращен ли элемент, прежде чем извлекать документ. Вы можете извлекать элементы только из библиотеки на сайте.  <br/> > **Внимание!**> При попытке извлечь элемент, который не был возвращен, происходит сбой рабочего процесса.           |
|Копировать документ  <br/> |Копирует документ из текущего списка в другой список библиотеки документов.  <br/> |
|Создать элемент в списке  <br/> |Создает новый элемент в указанной списке. В новом элементе можно указать поля и значения. Вы можете использовать это действие, когда требуется создать новый элемент с определенными сведениями.  <br/> |
|Удалить элемент  <br/> |Удаляет элемент.  <br/> > **Примечание**> Это действие завершается на компьютере, где запущена подсистема Workflow Manager, и возникает исключение **System.InvalidOperationException**. Обходного пути не существует.           |
|Отменить извлечение элемента  <br/> |Отменяет изменения и возвращает элемент, если он был извлечен и изменен.  <br/> > **Внимание!**> При попытке вернуть элемент, который не был извлечен, происходит сбой рабочего процесса.           |
|Задать поле в текущем элементе  <br/> |Задает указанное значение для указанного поля в текущем элементе.  <br/> > **Примечание**> Если рабочий процесс использовался для остановки при изменении значения поля, используйте действие **Дождаться события в элементе списка** вместо этого действия.          |
|Перевести документ  <br/> |Переводит документ на определенный язык  <br/> > **Примечание**> Необходимо заранее настроенное приложение-служба машинного перевода.           |
|Обновить элемент в списке  <br/> |Обновляет элемент списка. Вы можете задать поля и указать в них значения.  <br/> |
|Дождаться события в элементе списка  <br/> |[Расширенная версия действия Office 2010.] Останавливает текущий экземпляр рабочего процесса, чтобы дождаться указанного события для элемента списка. Это действие обнаруживает два события: **ItemUpdated** и **ItemAdded**.  <br/> |
|Ждать изменения поля в текущем элементе  <br/> |Ждет, когда поле текущего элемента будет иметь определенное значение.  <br/> |
   

### Действия для Project
<a name="bkm_ProjectActions"> </a>

Действия с проектами поддерживают интеграцию Microsoft Project. Они используются для создания рабочих процессов на основе Project. Все действия Project появились в SharePoint Designer 2013.
  
    
    

**Таблица 4. Справка по действиям Project**


|**Действие**|**Описание**|
|:-----|:-----|
|Создать проект из текущего элемента  <br/> |Берет текущий элемент и создает новый проект на сайте PWA фермы SharePoint.  <br/> |
|Задать поле проекта  <br/> |Задает значение определенного поля на сервере Project Server.  <br/> > **Примечание**> Для этого действия требуется сначала вернуть проект. Если проект не возвращен, рабочий процесс будет завершен, а пользователи не смогут открыть проект в Project Web App.           |
|Задать состояние стадии проекта  <br/> |Задает состояние стадии проекта.  <br/> > **Примечание**> Если текущий проект возвращен, возникает исключение.           |
|Задать поле состояния в списке идей  <br/> |Обновляет состояние исходного элемента списка, который сопоставлен с текущим проектом.  <br/> |
|Дождаться события проекта  <br/> |Ожидает определенного события проекта.  <br/> |
   

### Действия с задачами
<a name="bkm_TaskActions"> </a>

Действия с задачами позволяют вызывать рабочий процесс на основе платформы рабочих процессов SharePoint 2010 из рабочего процесса на основе платформы SharePoint 2013.
  
    
    

**Таблица 5. Справка по действиям с задачами**


|**Действие**|**Описание**|
|:-----|:-----|
|Назначить задачу  <br/> |Назначает пользователю задачу рабочего процесса и устанавливает конечный срок завершения проекта.  <br/> |
|Начать рабочий процесс  <br/> |Создает задачи для нескольких пользователей и позволяет проводить задачи через настраиваемый процесс.  <br/> |
   

### Вспомогательные действия
<a name="bkm_UtilityActions"> </a>

Вспомогательные действия  это действия, которые управляют строками или находят интервалы между датами.
  
    
    

**Таблица 6. Справка по вспомогательным действиям**


|**Действие**|**Описание**|
|:-----|:-----|
|Извлечь подстроку с конца строки  <br/> |Копирует указанное число символов, начиная с конца строки, и сохраняет результат в переменную.  <br/> |
|Извлечь подстроку, начиная с определенного символа строки  <br/> |Копирует подстроку, начиная с определенного символа строки, и заносит значение в переменную.  <br/> > **Примечание**> Помните, что хотя значения индекса в Microsoft SharePoint Designer 2013 начинаются с нуля, значения в SharePoint Designer 2010 были проиндексированы, начиная с 1.           |
|Извлечь подстроку с начала строки  <br/> |Копирует указанное число символов с начала строки и сохраняет результат в переменную.  <br/> |
|Извлечь подстроку указанной длины, начиная с определенного символа строки  <br/> |Копирует подстроку, состоящую из указанного количества символов, начиная с определенного символа строки, и заносит значение в переменную.  <br/> > **Примечание**> Помните, что хотя значения индекса в Microsoft SharePoint Designer 2013 начинаются с нуля, значения в SharePoint Designer 2010 были проиндексированы, начиная с 1.           |
|Найти интервал между датами  <br/> |Вычисляет интервал времени между двумя датами в минутах, часах или днях и сохраняет результат в переменную.  <br/> |
|Обрезать строку  <br/> |Удаляет пробелы в начале и конце строки.  <br/> |
|Найти подстроку в строке  <br/> |Находит определенную подстроку в строке и возвращает индекс ее начала.  <br/> |
|Заменить подстроку в строке  <br/> |Заменяет определенную подстроку другой подстрокой.  <br/> |
|Обрезать строку  <br/> |Удаляет пробелы в начале и конце строки.  <br/> |
